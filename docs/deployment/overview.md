# Deployment Overview

This section covers all deployment options for the CMA Case Management System, from cloud hosting to contributing compute resources.

## Available Deployment Options

### 1. AWS Deployment
Full-featured cloud deployment on AWS infrastructure with managed services.
- See [AWS Deployment Guide](aws-setup)

### 2. Cost-Optimized Deployment
Strategies for minimizing costs while maintaining performance.
- See [Cost Optimization Guide](cost-optimization)

### 3. Worker Deployment
Contribute compute resources by running CPU or GPU workers.
- See [Worker Deployment Guide](worker-deployment)

### 4. Monitoring & Maintenance
System monitoring, logging, and maintenance procedures.
- See [Monitoring Guide](monitoring)

## Quick Start

For most users, we recommend starting with the AWS deployment guide, which provides a comprehensive setup for production environments.

If you want to contribute compute resources to the distributed system, see the [Worker Deployment Guide](worker-deployment) for simple commands to get started.
